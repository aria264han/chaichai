# 拆拆 Showcase

拆拆 Showcase 是一个脱敏版项目展示仓库，用于展示一个多智能体短视频运营系统的架构设计、工作流拆解和代表性产出。

生产数据、内部配置、完整提示词、API Key、真实账号链接、企业微信 ID、完整自动化脚本和可直接复刻的执行链路均已移除或改写。本仓库重点展示系统能力，而不是交付生产代码。

## Project Summary

拆拆面向招聘短视频内容运营场景，支持从账号分析、视频拆片、脚本复刻、素材剪辑到知识库沉淀的完整流程。

系统关注的问题包括：

- 如何把优秀短视频拆解成可复用方法论
- 如何把跨行业视频结构迁移到招聘内容
- 如何让 AI 在多轮任务中保留长期业务经验
- 如何用多智能体分工处理拆片、视频生产、调试和知识归档
- 如何把一次性报告沉淀为可复用的知识库

## What Was Removed

为了保护业务资产和内部信息，展示版不包含：

- 真实账号链接、联系人、企业微信 ID
- 生产环境路径、私有配置、API Key
- 完整 `memory/` 和 `dreaming/` 原始记忆
- 完整 Wiki `sources/` 原始证据层
- 可直接运行的采集、下载、发送和定时任务脚本
- 全量拆片报告和账号报告
- 内部业务数据和未公开运营策略

## Core Capabilities

### 1. 多智能体分工

系统由主脑和多个子智能体组成：

| Role | Responsibility |
| --- | --- |
| 主脑 | 消息路由、任务调度、记忆管理、定时任务 |
| 拆片智能体 | 视频拆解、脚本复刻、分镜报告 |
| 视频智能体 | 素材整理、视频生成、剪辑交付 |
| 调试智能体 | 工具问题、模型边界、配置排障 |
| 知识库智能体 | 报告归档、知识分类、长期复用 |

### 2. 视频拆片

输入一条短视频链接和目标行业，系统可以输出：

- 逐帧分镜
- 视频数据分析
- 内容结构拆解
- 剪辑逻辑分析
- 可复用创作技巧
- 行业迁移脚本

### 3. 账号分析

系统支持按行业账号沉淀周期性报告，包括：

- 账号定位
- 粉丝与获赞变化
- 近期视频表现
- 爆款内容类型
- 选题风险
- 下一阶段内容建议

### 4. 知识库沉淀

项目将内容资产分为：

- 行业知识：岗位、薪资、用户画像、招聘痛点
- 方法论知识：标题、脚本、拍摄、剪辑、评论运营
- 综合报告：跨来源归纳后的阶段性结论
- 调试知识：工具、模型、API、配置问题

### 5. 从脚本到视频

展示版保留了流程设计：

```text
分镜表
→ 场景与素材需求解析
→ 口播、字幕、花字、转场参数化
→ 视频生成或素材剪辑
→ 输出短视频
```

生产版中具体 API、素材源、发送链路和自动化脚本未公开。
